ANSWER B:
**Recommendation**
Treat `SubtaskCurrentAttemptDetailsHandler` as the V1 runtime REST route for the current/latest subtask execution attempt:

- `GET /jobs/:jobid/vertices/:vertexid/subtasks/:subtaskindex`
- It is the non-attempt-numbered sibling of `.../attempts/:attempt`
- The docs anchor should be the generated V1 runtime REST API artifact, which is produced from the runtime REST headers and emitted as `rest_v1_dispatcher.html` by the docs build

**Implementation anchors**
