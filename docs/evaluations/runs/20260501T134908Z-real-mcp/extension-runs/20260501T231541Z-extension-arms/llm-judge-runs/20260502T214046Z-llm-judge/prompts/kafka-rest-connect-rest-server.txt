ANSWER B:
**Recommendation**
Use the 4.2.0 source comments and tests as the anchor, not a standalone docs page. The wiring story is:
1. `AbstractConnectCli.startConnect()` constructs `ConnectRestServer`, starts Jetty, then derives the worker ID from `advertisedUrl()`.
2. `Connect.start()` later calls `rest.initializeResources(herder)`.
3. `ConnectRestServer` supplies the public/admin resource sets and binds `Herder`, `RestClient`, and `RestServerConfig` into Jersey.
4. `RestServer` owns the generic Jetty/Jersey lifecycle and resource registration flow.
