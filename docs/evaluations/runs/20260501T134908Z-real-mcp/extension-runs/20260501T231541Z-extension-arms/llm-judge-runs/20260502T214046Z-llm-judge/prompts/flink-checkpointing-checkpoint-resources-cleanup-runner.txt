ANSWER B:
**Recommendation**
Use the 2.2.0 source Javadocs and the cleanup runner unit test as the primary anchors. The cleanup behavior is implemented by `CheckpointResourcesCleanupRunner`: it defers work until `closeAsync()`, then recreates the checkpoint store and counter, shuts them down with the derived job status, and closes `CheckpointsCleaner`. For docs, anchor the checkpoint-storage and externalized-checkpoint-retention config Javadocs, since this checkout does not include rendered markdown docs pages.
