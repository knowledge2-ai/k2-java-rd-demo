ANSWER B:
**Recommendation**
Use `CheckpointMetrics` as the canonical checkpoint-reporting payload in Flink 2.2.0: all nine metric fields are consumed by the reporting path, so the safe answer is that reporting uses every getter on `CheckpointMetrics`, with `unalignedCheckpoint` derived from `bytesPersistedDuringAlignment > 0` in the builder.

The fields used by reporting code are:
- `bytesProcessedDuringAlignment`
- `bytesPersistedDuringAlignment`
- `alignmentDurationNanos`
- `syncDurationMillis`
- `asyncDurationMillis`
- `checkpointStartDelayNanos`
- `unalignedCheckpoint`
- `bytesPersistedOfThisCheckpoint`
- `totalBytesPersisted`
