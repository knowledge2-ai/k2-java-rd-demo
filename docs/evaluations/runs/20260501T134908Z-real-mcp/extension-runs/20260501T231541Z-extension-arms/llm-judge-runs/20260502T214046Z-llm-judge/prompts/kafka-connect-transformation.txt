ANSWER B:
**Recommendation**
Anchor the behavior on the API contract in `Transformation.java`, then back it with the runtime path in `TransformationStage` and `TransformationChain`. For docs, the repo’s pinned source-of-truth is the `TransformationDoc` generator wired into `build.gradle`, since the rendered Connect SMT docs are generated from that code in the 4.2 tree.

**Implementation anchors**
- `connect/api/src/main/java/org/apache/kafka/connect/transforms/Transformation.java` defines the single-message contract: one record in, one record out, or `null` to drop; no mutation of reachable record state; thread-safe implementation.
- `connect/runtime/src/main/java/org/apache/kafka/connect/runtime/TransformationStage.java` is the per-stage adapter that applies an optional predicate, honors `negate`, and closes both predicate and transformation plugins.
- `connect/runtime/src/main/java/org/apache/kafka/connect/runtime/TransformationChain.java` is the serial executor for SMTs; it applies stages in order, stops on `null`, and routes execution through `RetryWithToleranceOperator`.
- For version-pinned docs generation, `connect/runtime/src/main/java/org/apache/kafka/connect/tools/TransformationDoc.java` plus the `genConnectTransformationDocs` task in `build.gradle` are the doc-producing anchors.

**Tests to inspect or add**

- `connect/runtime/src/test/java/org/apache/kafka/connect/runtime/TransformationConfigTest.java`
