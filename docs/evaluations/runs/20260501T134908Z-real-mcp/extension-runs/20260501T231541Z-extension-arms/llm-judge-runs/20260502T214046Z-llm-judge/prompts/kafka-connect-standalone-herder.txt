ANSWER B:
**Recommendation**
Use the standalone-mode source Javadocs plus `StandaloneHerder` itself as the anchor set. The behavior in Kafka Connect 4.2.0 is: validate the new config, stop the existing connector if replacing, write the config into the in-memory backing store, restart the connector, then recompute task configs and only restart tasks if the computed task config set changed. `patchConnectorConfig` is just a thin wrapper over that same replace path.
