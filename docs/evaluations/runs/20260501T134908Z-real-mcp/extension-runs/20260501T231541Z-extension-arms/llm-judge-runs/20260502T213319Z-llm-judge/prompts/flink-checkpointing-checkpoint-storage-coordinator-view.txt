ANSWER A:
**Recommendation**
`CheckpointStorageCoordinatorView` is the coordinator/admin half of Flink checkpoint storage. It defines what the JobManager-side checkpoint coordinator can do: check whether storage is HA-capable, check for a default savepoint location, resolve an external checkpoint/savepoint pointer, initialize base checkpoint directories, and create per-checkpoint/per-savepoint `CheckpointStorageLocation`s. `CheckpointStorageAccess` is the combined interface that adds the worker-side APIs.  
For a concrete implementation anchor, use the filesystem base class: `AbstractFsCheckpointStorageAccess`; it is the clearest implementation of the coordinator-facing contract in 2.2.0, with `FsCheckpointStorageAccess` and `MemoryBackendCheckpointStorageAccess` showing the concrete storage behavior.
