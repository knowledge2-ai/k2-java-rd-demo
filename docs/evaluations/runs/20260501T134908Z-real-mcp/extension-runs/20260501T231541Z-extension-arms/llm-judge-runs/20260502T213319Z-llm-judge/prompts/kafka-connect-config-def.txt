**Implementation anchors**
- `ConfigDef` is the shared config schema DSL: `define(...)` registers key metadata, type, default, validator, importance, docs, grouping, dependents, and recommender. Duplicate names are rejected. `repo://apache/kafka@4.2/clients/src/main/java/org/apache/kafka/common/config/ConfigDef.java:42` `:129` `:156`
- `parse(Map)` is strict: it throws on undefined dependents, missing required keys with no default, bad type conversions, and validator failures. It returns a typed `Map<String, Object>`. `repo://apache/kafka@4.2/clients/src/main/java/org/apache/kafka/common/config/ConfigDef.java:507` `:517` `:531` `:550`
- `validate(Map<String,String>)` is the softer path: it returns `ConfigValue` objects with parsed values, accumulated error messages, recommended values, and visibility. It does not stop at the first error. `repo://apache/kafka@4.2/clients/src/main/java/org/apache/kafka/common/config/ConfigDef.java:562` `:566` `:633` `:667`
- Type parsing is fixed by `Type` and `parseType(...)` for `BOOLEAN`, `STRING`, `INT`, `SHORT`, `LONG`, `DOUBLE`, `LIST`, `CLASS`, and `PASSWORD`. `repo://apache/kafka@4.2/clients/src/main/java/org/apache/kafka/common/config/ConfigDef.java:701`
- Connect builds on that with `ConnectorConfig.configDef()` and `Connector.validate(...)`, which combine base Connect settings with plugin/version recommenders and then call `ConfigDef.validate(...)`. `repo://apache/kafka@4.2/connect/runtime/src/main/java/org/apache/kafka/connect/runtime/ConnectorConfig.java:231` `:271` `:312` `repo://apache/kafka@4.2/connect/api/src/main/java/org/apache/kafka/connect/connector/Connector.java:137`
- Important Connect-specific nuance: `ConnectorConfig` uses validators/recommenders for connector class, converter class/version, transforms, predicates, and error settings. `repo://apache/kafka@4.2/connect/runtime/src/main/java/org/apache/kafka/connect/runtime/ConnectorConfig.java:241` `:246` `:252` `:292`
- List handling is nuanced: `parse()` deduplicates `ValidList` values and warns, while `validate()` surfaces duplicates as validation errors via `ValidList`. `repo://apache/kafka@4.2/clients/src/main/java/org/apache/kafka/common/config/ConfigDef.java:542` `:545` `:550` `:1017`
