ANSWER B:
**Recommendation**
Anchor the answer on the runtime REST API v1 docs and the concrete handler path. In Flink 2.2.0, `/overview` is served by `ClusterOverviewHandler`, which fetches a base `ClusterOverview` from the gateway and wraps it into `ClusterOverviewWithVersion` by injecting the build version and commit id.

**Implementation anchors**
- Version-pinned REST docs: `RuntimeRestAPIDocGenerator` generates the runtime docs only for `RuntimeRestAPIVersion.V1`; `V0` is test-only. That is the doc-version anchor for 2.2.0.
