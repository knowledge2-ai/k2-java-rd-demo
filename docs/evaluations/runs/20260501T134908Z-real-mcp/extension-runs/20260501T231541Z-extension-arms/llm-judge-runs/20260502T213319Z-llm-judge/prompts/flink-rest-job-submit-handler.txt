**Implementation anchors**
- The request-handling logic is in `JobSubmitHandler`: it indexes uploaded files by filename, rejects count mismatches, requires `executionPlanFileName`, deserializes the `ExecutionPlan`, uploads jars/artifacts to the blob server, submits the job, and returns `JobSubmitResponseBody("/jobs/" + jobId)` ([JobSubmitHandler.java](repo://apache/flink@release-2.2.0/flink-runtime/src/main/java/org/apache/flink/runtime/rest/handler/job/JobSubmitHandler.java#L57-L216)).
- The request contract is `JobSubmitRequestBody`: `executionPlanFileName` is the JSON field, with `jobGraphFileName` as an alias, plus jar and distributed-cache artifact lists ([JobSubmitRequestBody.java](repo://apache/flink@release-2.2.0/flink-runtime/src/main/java/org/apache/flink/runtime/rest/messages/job/JobSubmitRequestBody.java#L34-L77)).
- The endpoint contract is `POST /jobs`, multipart uploads enabled, with `202 Accepted` as the response status ([JobSubmitHeaders.java](repo://apache/flink@release-2.2.0/flink-runtime/src/main/java/org/apache/flink/runtime/rest/messages/job/JobSubmitHeaders.java#L29-L93)).
- The dispatcher wires this handler into the REST endpoint via `DispatcherRestEndpoint` ([DispatcherRestEndpoint.java](repo://apache/flink@release-2.2.0/flink-runtime/src/main/java/org/apache/flink/runtime/dispatcher/DispatcherRestEndpoint.java#L83-L99)).
- The versioned docs are generated from `RuntimeRestAPIDocGenerator` and `RuntimeOpenApiSpecGenerator` using `DocumentingDispatcherRestEndpoint`, which is the doc-generation view of the dispatcher REST surface ([RuntimeRestAPIDocGenerator.java](repo://apache/flink@release-2.2.0/flink-docs/src/main/java/org/apache/flink/docs/rest/RuntimeRestAPIDocGenerator.java#L69-L83), [RuntimeOpenApiSpecGenerator.java](repo://apache/flink@release-2.2.0/flink-docs/src/main/java/org/apache/flink/docs/rest/RuntimeOpenApiSpecGenerator.java#L45-L60)).
