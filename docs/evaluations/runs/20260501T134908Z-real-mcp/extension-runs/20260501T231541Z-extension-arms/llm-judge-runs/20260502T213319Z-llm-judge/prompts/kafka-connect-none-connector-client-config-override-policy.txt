ANSWER B:
**Recommendation**
- Treat `NoneConnectorClientConfigOverridePolicy` as a deny-all policy. It accepts an empty override map, but any requested client override is rejected during validation.
- The rejection is per config key, with the shared message shape: `The 'None' policy does not allow '<key>' to be overridden in the connector configuration.`

**Implementation anchors**
- Versioned Connect config docs are generated from `DistributedConfig` via `build.gradle`’s `genConnectConfigDocs` task, which writes `connect_config.html`: `repo://apache/kafka@4.2/build.gradle#L3664-L3668`
- The user-facing config doc for `connector.client.config.override.policy` is in `WorkerConfig.CONNECTOR_CLIENT_POLICY_CLASS_DOC`, which says `None` disallows overrides and `All` is the default: `repo://apache/kafka@4.2/connect/runtime/src/main/java/org/apache/kafka/connect/runtime/WorkerConfig.java#L159-L166`
- The API contract for validation is in `ConnectorClientConfigOverridePolicy` Javadoc: validation happens before connector startup, and any policy violation blocks startup: `repo://apache/kafka@4.2/connect/api/src/main/java/org/apache/kafka/connect/connector/policy/ConnectorClientConfigOverridePolicy.java#L26-L39`
- The concrete deny-all implementation is `NoneConnectorClientConfigOverridePolicy`, which returns `false` from `isAllowed(...)` for every config and logs that overrides are disallowed: `repo://apache/kafka@4.2/connect/runtime/src/main/java/org/apache/kafka/connect/connector/policy/NoneConnectorClientConfigOverridePolicy.java#L27-L47`

**Tests to inspect or add**
- `NoneConnectorClientConfigOverridePolicyTest` is the direct unit anchor: it asserts empty overrides pass and a map with `SaslConfigs.SASL_JAAS_CONFIG` plus `ProducerConfig.ACKS_CONFIG` fails: `repo://apache/kafka@4.2/connect/runtime/src/test/java/org/apache/kafka/connect/connector/policy/NoneConnectorClientConfigOverridePolicyTest.java#L28-L49`
- `BaseConnectorClientConfigOverridePolicyTest` is the shared harness that turns a client-config map into a `ConnectorClientConfigRequest` and checks for per-entry error messages: `repo://apache/kafka@4.2/connect/runtime/src/test/java/org/apache/kafka/connect/connector/policy/BaseConnectorClientConfigOverridePolicyTest.java#L29-L59`
- The sibling policy tests are useful context for expected behavior differences: `AllowlistConnectorClientConfigOverridePolicyTest` and `PrincipalConnectorClientConfigOverridePolicyTest`: `repo://apache/kafka@4.2/connect/runtime/src/test/java/org/apache/kafka/connect/connector/policy/AllowlistConnectorClientConfigOverridePolicyTest.java#L32-L75` and `repo://apache/kafka@4.2/connect/runtime/src/test/java/org/apache/kafka/connect/connector/policy/PrincipalConnectorClientConfigOverridePolicyTest.java#L28-L49`
- `WorkerTest` is the runtime enforcement anchor: it verifies `None` causes `Worker.baseConsumerConfigs(...)` and `Worker.adminConfigs(...)` to throw `ConnectException` when connector overrides are present: `repo://apache/kafka@4.2/connect/runtime/src/test/java/org/apache/kafka/connect/runtime/WorkerTest.java#L1208-L1222` and `repo://apache/kafka@4.2/connect/runtime/src/test/java/org/apache/kafka/connect/runtime/WorkerTest.java#L1255-L1267`
