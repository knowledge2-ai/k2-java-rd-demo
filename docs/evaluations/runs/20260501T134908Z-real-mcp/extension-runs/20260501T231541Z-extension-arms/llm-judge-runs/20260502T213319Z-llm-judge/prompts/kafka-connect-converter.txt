ANSWER B:
**Recommendation**
Use the `Converter` interface as the API contract, but anchor the key/value behavior on the runtime wiring and the versioned config docs, not on the interface alone. The most useful concrete implementation anchor is `StringConverter`, with `NumberConverter` and `ByteArrayConverter` as sibling examples.

**Implementation anchors**
- `Converter` defines the split surface: `configure(Map, boolean isKey)`, plus separate `fromConnectData`/`toConnectData` methods and header-aware overloads.
- `WorkerConfig` and `ConnectorConfig` carry the version-pinned docs for both key and value converters via `key.converter.plugin.version` / `value.converter.plugin.version`.
- `Plugins.newConverter(...)` is where Connect decides key vs value by config name and passes `isKey` into the converter.
- `StringConverter` is the clearest concrete example of key/value handling: it maps `isKey` into `converter.type` and then configures the serializer/deserializer accordingly.
- `NumberConverter` follows the same pattern for numeric converters; `ByteArrayConverter` is the pass-through counterexample and ignores key/value distinction.

**Tests to inspect or add**
- `connect/runtime/src/test/java/org/apache/kafka/connect/converters/NumberConverterTest.java` and the concrete number tests (`LongConverterTest`, `DoubleConverterTest`, etc.) for round-trip, null, and type-mismatch behavior.
- `connect/runtime/src/test/java/org/apache/kafka/connect/converters/ByteArrayConverterTest.java` for schemaless bytes, invalid schema/type, and `ByteBuffer` handling.
- `connect/runtime/src/test/java/org/apache/kafka/connect/runtime/AbstractWorkerSourceTaskTest.java` and `WorkerSourceTaskTest.java` for source-side key vs value serialization paths.
- `connect/runtime/src/test/java/org/apache/kafka/connect/runtime/WorkerSinkTaskTest.java` for sink-side key vs value deserialization and plugin wrapping.
- `connect/runtime/src/test/java/org/apache/kafka/connect/runtime/WorkerTest.java` and `connect/runtime/src/test/java/org/apache/kafka/connect/runtime/isolation/MultiVersionTest.java` for versioned key/value converter loading from connector vs worker configs.
