ANSWER B:
**Recommendation**
- Anchor the answer on `JobCancellationHandler` itself, not a `/cancel`-suffixed route. In 2.2.0 the REST contract is `PATCH /jobs/:jobid` with optional `mode`; the endpoint wiring defaults to `CANCEL`, and `STOP` is retained only as legacy redirect behavior.
- State the response contract explicitly: success returns `202 Accepted` with `EmptyResponseBody`; errors are mapped to `409 Conflict` for already-terminal jobs, `408 Request Timeout`, `404 Not Found`, and `500 Internal Server Error`.
